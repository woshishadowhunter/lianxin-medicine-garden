# Security Policy

## Supported Versions

The `main` branch is the currently supported open source version.

## Reporting a Vulnerability

Please avoid opening a public issue for vulnerabilities that expose credentials, user data, administrator access, cloud storage paths, or database permission problems.

If you find a security issue, contact the maintainer through GitHub profile contact methods or open a minimal public issue that says a private security report is needed, without including exploit details.

## Security Areas To Review

- Administrator login and role handling
- Cloud database permission rules
- Cloud function input validation
- Photo upload paths and preview permissions
- Data export authorization
- Seed scripts and administrator password handling

## Public Demo Data

Do not commit real family names, phone numbers, OpenIDs, cloud file IDs, or private community deployment data. Use synthetic data only.
