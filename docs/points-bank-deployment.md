# 连心药园积分银行部署说明（零基础版）

这份说明按“第一次部署微信小程序”的水平编写。请从上往下操作，不要跳步骤。

> [!CAUTION]
> 你的项目已经有家庭、药材和养护记录，因此属于“已有数据环境”。
> **不要执行 `init-database` 云函数，也不要重新初始化数据库。**
> 它会再次生成 200 个家庭和种植任务，可能造成重复数据。

## 一、部署前先确认三件事

### 1. 准备微信开发者工具

需要安装微信开发者工具，并使用有该小程序开发权限的微信扫码登录。

本项目的小程序 AppID 已经写在 `project.config.json` 中：

```text
your-wechat-miniprogram-appid
```

如果打开项目时提示“没有权限”，不是代码问题，需要让小程序管理员在微信公众平台把你的微信添加为开发者。

### 2. 打开正确的项目目录

在微信开发者工具中点击“导入项目”或“打开项目”，选择：

```text
E:\项目内容\本草种植小程序
```

不要只选择 `miniprogram` 文件夹，也不要只选择 `cloudfunctions` 文件夹。

打开后，左侧文件树应该同时看到：

```text
miniprogram
cloudfunctions
docs
```

### 3. 确认云环境一致

项目代码当前使用的云环境 ID 是：

```text
your-cloud-env-id
```

操作方法：

1. 在微信开发者工具顶部点击“云开发”。
2. 查看当前云环境名称或环境 ID。
3. 确认环境 ID 是你自己的云开发环境 ID。
4. 如果顶部可以切换环境，选择这个环境。

> [!WARNING]
> 不要为了部署积分系统新建另一个云环境。新环境中没有原来的家庭和养护数据。

如果云开发控制台中的环境 ID 和上面不同，请先停止操作，不要继续部署。需要先确认原有数据到底存在哪个环境。

---

## 二、先备份重要数据（建议执行）

积分系统不会主动删除原数据，但第一次操作时建议备份。

1. 点击开发者工具顶部“云开发”。
2. 进入“数据库”。
3. 找到 `families` 集合，点击“导出”并保存文件。
4. 找到 `care_records` 集合，点击“导出”并保存文件。
5. 找到 `admins` 集合，点击“导出”并保存文件。

不同版本的开发者工具中，“导出”可能位于集合右侧的“更多”菜单。

只要这三个集合已经备份，就可以继续。

---

## 三、创建积分系统需要的数据库集合

“集合”可以理解为数据库中的表。

在云开发控制台中：

1. 点击“数据库”。
2. 点击集合列表旁边的“+”或“添加集合”。
3. 按照下面的名称逐个创建。

### 集合 1：家庭积分账户

```text
points_accounts
```

### 集合 2：积分流水

```text
points_transactions
```

### 集合 3：家庭成员绑定关系

```text
family_members
```

名称必须完全一致，全部使用英文小写和下划线，不能有空格。

创建完成后，数据库左侧应该能看到：

- `points_accounts`
- `points_transactions`
- `family_members`

### 设置集合权限

这三个集合都只允许通过云函数访问，不应该让小程序前端直接修改。

对三个集合分别进行以下操作：

1. 打开集合。
2. 找到“权限设置”或“数据权限”。
3. 选择最严格的客户端权限，例如“所有用户不可读写”或“仅管理端可读写”。
4. 保存。

云函数仍然可以正常访问这些集合。

> [!CAUTION]
> 不要设置成“所有用户可读写”，否则普通用户可能绕过积分规则直接修改数据。

## 四、创建数据库索引

索引相当于数据库的目录。缺少索引时，积分流水或历史补发查询可能报错。

### 索引 1：家庭积分流水

1. 打开 `points_transactions` 集合。
2. 点击“索引管理”。
3. 点击“新建索引”。
4. 索引名称可以填写：

```text
family_points_history
```

5. 添加两个字段：

| 字段 | 排序 |
|---|---|
| `family_code` | 升序 |
| `created_at` | 降序 |

6. 点击“确定”或“创建”。

### 索引 2：历史积分补发

1. 打开原有的 `care_records` 集合。
2. 进入“索引管理”。
3. 点击“新建索引”。
4. 索引名称可以填写：

```text
points_backfill
```

5. 添加三个字段：

| 字段 | 排序 |
|---|---|
| `audit_status` | 升序 |
| `points_status` | 升序 |
| `created_at` | 升序 |

6. 点击“确定”或“创建”。

索引创建后可能需要几十秒到几分钟才会生效。状态显示“可用”或“已启用”后再继续。

### 其他索引

`admins.openid` 和普通的 `created_at` 单字段查询通常可以使用系统默认索引。如果运行时明确提示缺少某个索引，再根据错误提示创建即可。

## 五、部署四个云函数

云函数必须部署到第二步确认过的云环境中。

需要依次部署：

1. `login`
2. `pointsBank`
3. `auditReview`
4. `exportData`

每个云函数都按下面的方法操作。

### 部署 `login`

1. 在开发者工具左侧找到：

```text
cloudfunctions/login
```

2. 右键点击 `login` 文件夹。
3. 选择“上传并部署：云端安装依赖”。
4. 等待底部控制台提示上传成功。

### 部署 `pointsBank`

1. 右键点击：

```text
cloudfunctions/pointsBank
```

2. 选择“上传并部署：云端安装依赖”。
3. 等待成功提示。

### 部署 `auditReview`

1. 右键点击：

```text
cloudfunctions/auditReview
```

2. 选择“上传并部署：云端安装依赖”。
3. 等待成功提示。

### 部署 `exportData`

1. 右键点击：

```text
cloudfunctions/exportData
```

2. 选择“上传并部署：云端安装依赖”。
3. 等待成功提示。

> [!IMPORTANT]
> 必须选择“云端安装依赖”，不要只选择“上传”。云函数依赖 `wx-server-sdk`。

### 检查是否部署成功

1. 打开“云开发”。
2. 点击“云函数”。
3. 确认列表中存在 `login`、`pointsBank`、`auditReview` 和 `exportData`。
4. 打开每个云函数，确认最近更新时间是本次部署时间。
5. 确认 `pointsBank` 和 `auditReview` 的超时时间为 60 秒。如果控制台显示的仍是旧值，重新部署一次。

> [!CAUTION]
> 当前是已有数据环境，**不要运行 `init-database`**。

## 六、编译小程序前端

云函数部署后，回到开发者工具主界面：

1. 点击顶部“编译”。
2. 等待模拟器重新打开首页。
3. 点击底部“我的”。
4. 已绑定家庭时，应看到“连心积分银行”入口。

如果编译后没有积分入口：

1. 点击顶部“清缓存”。
2. 选择“清除文件缓存”或“清除编译缓存”。
3. 再点击“编译”。

不要随意选择“清除全部数据缓存”，否则模拟器中保存的家庭绑定状态会被清除，需要重新绑定。

---

## 七、重新登录管理员（这一步不能省略）

新版本会把当前管理员的微信身份与云端管理员记录绑定。如果不重新登录，发积分时会提示管理员身份失效。

### 如果当前已经是管理员模式

1. 打开“我的”。
2. 进入“数据总览”。
3. 点击管理后台顶部的“退出”。
4. 返回“我的”。

### 重新登录

1. 在“我的”页面找到“切换为管理员”。
2. 点击“验证”。
3. 输入管理员密码。
4. 登录成功后进入“数据总览”。

初始化脚本中的默认超级管理员密码是：

```text
your-strong-admin-password
```

如果你之前修改过密码，请使用修改后的密码。

登录成功后，`admins` 集合中对应管理员记录会自动增加：

- `openid`
- `last_login`

可以在云开发数据库中打开 `admins` 集合确认这两个字段是否出现。

> [!WARNING]
> 正式上线前建议修改默认管理员密码，不要长期使用示例密码。

## 八、补发以前已经审核通过的积分

这个步骤用于补发积分系统上线前的历史养护记录。

1. 进入小程序“我的”。
2. 在管理员分组中点击“积分银行”。
3. 页面如果出现黄色提示“历史积分待补发”，点击“补发一批”。
4. 每次最多处理 20 条。
5. 等待“已补发 XX 条”的提示。
6. 如果黄色提示仍然存在，继续点击“补发一批”。
7. 重复操作，直到黄色提示消失或待补发数量变成 0。

补发过程可以安全重复。每条养护记录使用固定交易编号，重复点击不会重复加分。

补发完成后：

- `points_accounts` 中会出现家庭积分账户。
- `points_transactions` 中会出现积分流水。
- `care_records` 中已补发的记录会增加 `points_status: awarded`。

## 九、按顺序验收功能

不要一上来测试所有功能。按下面顺序逐项确认。

### 测试 1：家庭积分页面

1. 打开“我的”。
2. 点击“连心积分银行”。
3. 确认页面能显示余额、累计入账、交易笔数和流水。
4. 页面底部应显示积分规则。

### 测试 2：审核自动发 10 分

1. 用家庭账号新提交一条带照片的养护记录。
2. 切换为管理员。
3. 进入“审核中心”。
4. 对这条记录点击“确认通过”。
5. 系统应提示发放积分。
6. 返回该家庭的积分账户，余额应增加 10。
7. 流水中应出现“养护记录审核通过”。

### 测试 3：审核退回自动冲正

1. 管理员在审核中心切换到“已确认”。
2. 找到刚才的记录。
3. 将它标记为“需修正”。
4. 家庭积分余额应减少 10。
5. 流水中应出现“积分冲正”。

### 测试 4：管理员发优秀种植奖励

1. 进入“我的 → 管理员 → 积分银行”。
2. 在“家庭账户”中选择一个测试家庭。
3. 奖励类型选择“种植效果优秀（+50）”。
4. 奖励说明可以填写：

```text
薄荷长势良好，档案照片完整
```

5. 点击“确认发放积分”。
6. 在二次确认弹窗中再次确认。
7. 家庭余额应增加 50，并产生奖励流水。

### 测试 5：导出积分流水

1. 进入管理后台“数据导出”。
2. 选择“积分银行流水”。
3. 点击“导出 Excel”。
4. 打开导出的文件。
5. 文件应包含两个 Sheet：

- 积分账户
- 积分流水

以上五项都通过，积分系统才算部署成功。

## 十、上传体验版或正式版本

云函数部署成功只代表后端已更新，还需要上传小程序前端，手机用户才能看到积分页面。

### 上传代码

1. 在微信开发者工具中再次点击“编译”，确认没有红色报错。
2. 点击右上角“上传”。
3. 填写版本号，例如：

```text
1.1.0
```

4. 填写项目备注，例如：

```text
新增积分银行、养护积分、优秀种植奖励和积分流水
```

5. 点击“上传”。
6. 等待提示上传成功。

### 设置为体验版

1. 登录微信公众平台的小程序后台。
2. 进入“管理 → 版本管理”。
3. 在“开发版本”中找到刚上传的版本。
4. 选择“设为体验版”。
5. 使用体验成员微信扫码测试。

### 提交正式审核

体验版全部测试通过后：

1. 在“版本管理”的开发版本中点击“提交审核”。
2. 按页面要求填写功能页面和测试说明。
3. 等待微信审核。
4. 审核通过后点击“发布”。

不要在没有完成第九部分验收前直接发布正式版。

## 十一、常见报错处理

### 报错：云函数不存在 / function not found

原因：`pointsBank` 没有部署，或者部署到了错误云环境。

处理：

1. 确认环境 ID 是你自己的云开发环境 ID。
2. 右键 `cloudfunctions/pointsBank`。
3. 重新选择“上传并部署：云端安装依赖”。

### 报错：collection not exists / 集合不存在

原因：数据库集合名称没有创建或拼写错误。

处理：确认存在以下三个集合：

```text
points_accounts
points_transactions
family_members
```

### 报错：query requires an index / 查询需要索引

原因：第四部分的索引没有创建或仍在构建。

处理：

1. 复制错误信息中给出的索引字段。
2. 返回云开发数据库的“索引管理”。
3. 按第四部分重新创建索引。
4. 等待索引状态变为可用。

### 报错：管理员身份已失效，请重新登录

原因：管理员记录还没有绑定当前微信的 `openid`，或者 `login` 云函数仍是旧版本。

处理：

1. 重新部署 `login`。
2. 退出管理员模式。
3. 重新输入管理员密码登录。
4. 检查 `admins` 集合是否出现 `openid` 字段。

### 积分页面能打开，但余额一直是 0

可能原因：

- 还没有审核通过的记录。
- 没有执行历史积分补发。
- `auditReview` 仍是旧版本。

处理：

1. 重新部署 `auditReview`。
2. 在积分银行中执行历史补发。
3. 新审核一条测试记录。
4. 检查 `points_transactions` 是否产生流水。

### 审核成功，但没有增加积分

处理顺序：

1. 打开云开发控制台。
2. 进入“云函数 → auditReview → 日志”。
3. 查看最新日志中的错误。
4. 确认 `points_accounts` 和 `points_transactions` 集合存在。
5. 确认索引已生效。
6. 确认管理员已重新登录。

### 历史补发每次只处理 20 条

这是正常设计，不是故障。分批处理可以防止云函数超时。继续点击“补发一批”直到数量为 0。

### 页面提示没有权限

处理：

1. 确认当前微信是小程序开发者或管理员。
2. 确认云环境选择正确。
3. 确认数据库集合权限设置为客户端不可直接写入。
4. 积分数据必须通过云函数操作，不要在前端直接修改。

---

## 十二、绝对不要做的操作

- 不要在现有环境再次运行 `init-database`。
- 不要手动修改 `points_accounts.balance`。
- 不要删除 `points_transactions` 中的流水来“改余额”。
- 不要把积分集合设置成所有用户可写。
- 不要把云函数部署到另一个空云环境。
- 不要在历史补发过程中同时大批量审核记录。
- 不要直接在正式用户数据上测试，先使用一个测试家庭。
